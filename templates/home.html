Current Work: Federated Learning for Fraud Detection

Our system leverages federated learning to enhance credit card fraud detection. Federated learning is an innovative machine learning approach where multiple institutions collaboratively train a shared model without sharing their local data. Each participating bank trains their local model on their own data, and only the learned model parameters are shared with the central server. This approach aggregates the knowledge from all participating banks to create a more robust and accurate fraud detection system.

By keeping sensitive data localized and only sharing model updates, federated learning not only improves the prediction accuracy but also ensures that customer data remains secure and private. This distributed learning method helps in overcoming data silos and enhances the model's performance by incorporating diverse data sources while respecting privacy regulations.

How to Use the API Request

Each registered bank can make predictions by sending a POST request to our API. Below are examples of how to use `curl` to send a request. Replace `` with the bank's password and adjust the feature values as needed. 

curl -X POST http://your-server-url/predict \
    -H "Content-Type: application/json" \
    -d '{"password": "", "features": [0.186, 0.223, 0.336, 0.0, 1.0, 0.0, 1.0]}'

This `curl` command sends a JSON payload with the bank's password and feature values to the prediction endpoint. The response will include the prediction result, indicating whether the transaction is classified as fraud or not.
