Mechanism for Federated Learning between Banks

Local Models for Each Bank

Each bank (Bank1, Bank2, Bank3) has its own local model:

  • Bank1: Local Model1
  • Bank2: Local Model2
  • Bank3: Local Model3

These models are trained locally within each bank's environment using their private data.

Accessing the Global Model

When a bank needs to access the global model, it makes an API request using its own password for authentication.

  • Bank1: Password1
  • Bank2: Password2
  • Bank3: Password3

The global model aggregates the updates from each bank's local model to improve fraud detection across all banks.

Data Flow and Privacy

Each bank sends only model updates, maintaining data privacy. They do not share any sensitive data, ensuring compliance with data privacy regulations.

This federated approach ensures that the global model benefits from each bank's data while keeping the individual data secure and private.
